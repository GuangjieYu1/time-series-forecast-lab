# v0.4 Transparent Time Series Experiment Platform

## 定位

v0.4 停止横向新增模型，目标是把预测工具收敛为透明时间序列实验平台。每次实验必须满足：

- Observable：阶段、模型、资源、优化和日志可实时观察。
- Explainable：Feature 生命周期、重要性、SHAP、残差与数据健康可解释。
- Reproducible：数据、配置、代码、环境和随机种子可校验并可复现。
- Benchmarkable：Clean、Dirty、Edge、Large 使用统一结果契约持续比较。
- Extensible：新增模型只实现后端能力，不要求修改前端数据结构。

## 范围约束

v0.4 不新增预测模型，不接 Chronos、Moirai、PatchTST 或 N-BEATS。已有计划中模型继续只作为模型注册表元数据展示。DeepSeek 总结位于 P2，不能早于 P0 平台闭环。

## 实施顺序

### P0-A 平台契约

统一并版本化：

- Experiment Manifest
- Feature Pipeline
- Covariate Descriptor
- Runtime Event
- Benchmark Result
- History Snapshot

新实验写入 Manifest v0.4；历史 Manifest v0.3 保持可读取。Runtime、Feature Factory、历史详情、Benchmark 和报告必须读取同一份实验产物。

### P0-B Feature Factory 与 Covariate Flow

Feature Factory 由 Summary、Pipeline Timeline、Feature Machines、Feature Flow Graph、Feature Detail Drawer 组成。页面展示 Generator 和数据流，不退化为 Feature 列表。

协变量只支持：

- Known Future：Backtest 使用 future timeline，Forecast 使用已提供未来值。
- Static：Backtest 和 Forecast 都 repeat last known。

禁止读取测试集真实未来协变量。每个 Feature 记录 Generated、Selected、Dropped、Importance、SHAP，以及 Drop Reason。

### P0-C Model Console

统一 RuntimeEvent 驱动：

`Loading -> Feature Engineering -> Training -> Forecast -> Residual -> Finished`

每个模型展示 Status、Stage、Elapsed、Remaining、CPU、Memory、GPU、Current Trial、Current Metric、Warnings 和 Logs。实时运行和历史回放使用同一事件结构。

### P0-D Benchmark

目录固定为：

```text
backend/benchmarks/fixtures/
  clean/
  dirty/
  edge/
  large/
```

覆盖空时间、非法时间、Excel Serial Date、科学计数法日期、中文日期、重复时间、乱序时间、文本目标、异常值、全空 Sheet、说明行，以及 10k、100k、500k 数据规模。

统一输出 Success Rate、Failure Rate、Warnings、Runtime、Memory 和 Residual Metrics。Benchmark 结果必须通过 Pydantic Schema 校验。

### P0-E 实验复现

Manifest 必须包含：

- Dataset Hash
- Config Hash
- Model Version
- Feature Pipeline Version
- Runtime Event Schema Version
- Random Seed
- Environment
- Git Commit
- Python Version

复现实验要求用户重新上传文件并校验 Dataset Hash；系统不保存原始文件。

### P1

- Optimization Engine：Optuna TPE、Median Pruner、Successive Halving 和实时 Trial。
- Runtime Learning：使用 Benchmark 与历史真实运行持续校准估时，并记录预测误差。
- Explainability：Feature Importance、真实 SHAP、Residual Diagnosis、Model Health、Data Health、Outlier Detection。

### P2

- 深色实验室视觉统一与动画优化。
- Feature Flow、Runtime、实验历史时间轴体验优化。
- DeepSeek v4 Flash 一键中文总结。

## Definition of Done

P0 完成必须同时满足：

1. 同一次实验的实时页与历史页展示相同 Runtime、Feature 和模型结果。
2. Manifest v0.4 可下载，v0.3 历史记录仍可读取。
3. Feature Factory 展示真实生成、选择和丢弃结果，不使用静态示例。
4. Known Future 与 Static 协变量通过防泄漏测试。
5. 单模型失败不会中断事件流、排行榜或历史保存。
6. Benchmark 覆盖四类数据并输出统一可校验结果。
7. 原始上传文件不落库，复现依赖重新上传并校验 Hash。
8. 后端全量 pytest、前端 typecheck 和 build 均通过。

## 当前基础

仓库已经具备 Feature Factory、Runtime Inspector、Covariate Flow、Optuna、Benchmark、Manifest 和历史回放原型。v0.4 的工作重点是消除重复契约和静态推导，让这些原型共享真实、持久化、可版本化的实验事实。
